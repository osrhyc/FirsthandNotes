> 原书：*Inside the Black Box*（Rishi K. Narang，第二版），量化基金 Telesis Capital 创始人。多份中文书单公认的"量化第一本书"——不需要数学基础，读完能建立完整的量化世界观。
>
> **关于本精读**：原创读书笔记，**每章一页**（第四部分四章合为一页），帮你按原书结构逐章消化；细节与作者原话以原书为准。

## 这本书解决什么问题

市面上讲量化的材料要么是新闻式猎奇（"神秘的赚钱机器"），要么是论文式硬核（一页十个公式）。这本书填的正是中间的空白：**用普通人能懂的语言，把量化交易系统的内部结构一个模块一个模块讲清楚。**

读完你能回答三个问题：量化系统由哪几个部件组成？评价一个量化策略该问哪些问题？量化会在什么情况下亏大钱？

## 全书一张图（先记住它）

作者最大的贡献，是指出**所有量化交易系统都是同一张结构图**：

```
                ┌─────────────────────────────┐
                │        研究（Research）        │
                │     数据（Data）— 燃料        │
                └──────────────┬──────────────┘
                               ↓
   ┌───────────┐  ┌───────────┐  ┌─────────────┐
   │ Alpha 模型 │  │  风险模型  │  │ 交易成本模型  │
   │ (赚钱观点) │  │ (控制暴露) │  │ (交易的代价)  │
   └─────┬─────┘  └─────┬─────┘  └──────┬──────┘
         └───────────── ↓ ──────────────┘
                ┌───────────────┐
                │  组合构建模型   │  ← 三方意见的"裁判"
                └───────┬───────┘
                        ↓
                ┌───────────────┐
                │   执行模型     │  → 下单成交
                └───────────────┘
```

一句话：**Alpha 模型提出"买什么"，风险模型说"别太集中"，成本模型说"别换太勤"，组合构建模型综合三方拍板，执行模型负责落地；数据是燃料，研究是引擎**。读每一章时都回头看看它在图中的位置。
