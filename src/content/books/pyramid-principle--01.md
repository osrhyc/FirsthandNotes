## 先读结论

1. 这本书最适合解决一个问题：资料很多、想法很多，但写出来没有主线。
2. 金字塔结构的核心不是排版，而是思考顺序：结论先行，以上统下，归类分组，逻辑递进。
3. 对读者来说，最容易理解的顺序是先看到主要观点，再看到支撑观点。文章难读，常常不是句子差，而是思想顺序和读者理解顺序冲突。
4. 纵向关系是疑问/回答：上层观点让读者产生问题，下层观点直接回答它。
5. 横向关系是同组逻辑：同一组里的观点必须属于同一范畴，并按时间、结构、程度等顺序排列。
6. 序言不是装饰，它负责让读者知道：背景是什么、冲突是什么、问题是什么、答案是什么。
7. 写作时不要只列“5 个问题”“10 个步骤”。如果这些条目没有共同逻辑，就只是罗列，不是结构。

## 关键摘取

### 1. 先写答案，再写解释

这本书最可迁移的一句话是：先让读者知道你的中心思想，再展开支持材料。

可以把任何文章先压缩成这个模板：

```text
我想说的结论是：____。
我有三个理由：
1. ____。
2. ____。
3. ____。
所以你应该：____。
```

如果这个模板填不出来，不要急着写正文，先回到结构。

### 2. 纵向关系：每一层都回答上一层的问题

金字塔不是目录树，而是一连串问答。顶层观点抛出读者疑问，下一层直接回答。

例子：

```text
结论：这篇文章应该先重写结构。
读者问：为什么？
回答：主旨不清、论点混杂、案例没有服务结论。
读者问：怎么改？
回答：先定中心思想，再重排论点，最后删掉无关案例。
```

检查方法：每个二级标题都问一句“它在回答上级标题的什么问题？”答不上来，就说明结构断了。

### 3. 横向关系：同一组必须同类

第 2 篇提醒，一个常见错误是把一些能用同一个名词概括的东西放在一起，但它们之间其实没有逻辑关系。真正的分组要满足两个条件：

- 属于同一逻辑范畴。
- 按某种顺序排列：时间顺序、结构顺序、程度顺序。

这对你的文章管理很有用。比如“量化入门”不能随便列：Python、基金、回测、心态、券商开户。更好的分组是：

| 组 | 内容 |
| --- | --- |
| 市场对象 | 股票、基金、指数、ETF |
| 数据与工具 | 数据源、清洗、Python、回测框架 |
| 策略研究 | 假设、因子、组合、风险 |
| 实盘约束 | 成本、流动性、交易纪律 |

### 4. 序言用 SCQA 引出问题

书里把序言拆成背景、冲突、疑问、回答。它的作用不是铺垫气氛，而是把读者带到同一个问题现场。

写文章时可以这样问：

| 元素 | 问题 |
| --- | --- |
| 背景 | 读者已经知道或认同什么？ |
| 冲突 | 现在出现了什么变化、矛盾或麻烦？ |
| 疑问 | 读者自然会问什么？ |
| 回答 | 我的中心结论是什么？ |

这个模板可以直接加入写作 agent：每篇长文先生成 SCQA，再生成大纲。

### 5. 先结构，后表达

作者在前言中说，很多写作问题不在句子，而在思想组织。对普通写作者来说，这个提醒很重要：不要一上来润色句子，先检查中心思想、分组和顺序。

修改文章可以按这个顺序：

1. 一句话说出中心思想。
2. 列出支撑它的 3-5 个关键句。
3. 检查每组是否同类。
4. 检查每组是否有顺序。
5. 最后才改标题、句子和过渡。
