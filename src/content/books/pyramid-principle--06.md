> TL;DR：全书可以压缩成一个动作：先替读者把思想关系整理好，再表达。金字塔原理不是“总分总”这么简单，而是一套检查思想的规则：中心思想是否明确，下层是否支撑上层，同组是否同类，顺序是否有逻辑，序言是否引出读者问题。它最适合用于文章大纲、学习文档、咨询报告、PPT 和 AI 写作前的结构校验。

## 详读

### 全书复习卡

| 模块 | 核心问题 | 关键工具 |
| --- | --- | --- |
| 表达 | 读者如何理解我？ | 结论先行、疑问/回答、SCQA |
| 思考 | 我的分组是否有意义？ | 时间顺序、结构顺序、程度顺序 |
| 解决问题 | 我到底在解决什么？ | 现状、目标、差距、逻辑树 |
| 演示 | 结构如何被看见？ | 标题层级、过渡句、图表问题 |

### 这本书的边界

《金字塔原理》非常适合结构化表达，但它不负责内容本身的真实性。一个结构漂亮的论证，如果事实错了、证据弱了，仍然不成立。

所以在本项目里，它应该和“研究优先”和“真读原书”配合使用：

- 真读原书保证材料来源；
- 研究优先保证事实核查；
- 金字塔原理保证表达结构；
- 卡片笔记法保证长期积累。

### 给写作 agent 的规则

每篇长文起草前，先生成结构检查项：

1. 中心思想：一句话能说清吗？
2. 读者疑问：读者为什么需要读？
3. 关键句：3-5 个理由是否支撑中心思想？
4. 横向关系：这些理由是否同类，并按顺序排列？
5. 证据：每个理由下面是否有事实、案例或数据？
6. 序言：是否包含背景、冲突、疑问、回答？

如果这些问题回答不好，不应该直接写正文。

### 行动清单

- [ ] 给写作 agent 增加 SCQA 大纲步骤。
- [ ] 给每篇文章增加“中心思想”和“关键句”内部草稿。
- [ ] 修改长文时先改结构，再改句子。
- [ ] 对学习文档使用“目标能力 -> 步骤 -> 示例 -> 练习”的金字塔。
- [ ] 对内容拆解使用“中心论点 -> 支撑理由 -> 证据 -> 漏洞”的金字塔。

## 要点清单

- 结构清晰来自思想关系清晰。
- 金字塔原理是写作前的思考工具，也是写作后的检查工具。
- 结论先行不是粗暴先说答案，而是先回答读者最关心的问题。
- 罗列没有用，必须概括出上层思想。
- 这本书应该成为项目里所有长文生成的结构规范。
