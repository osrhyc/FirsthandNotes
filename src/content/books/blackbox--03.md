## 模块分工表

| 模块 | 回答的问题 | 角色 |
| --- | --- | --- |
| Alpha 模型 | 什么会涨、什么会跌？ | 进攻 |
| 风险模型 | 哪些风险我不想承担？ | 防守 |
| 交易成本模型 | 动一次仓要花多少钱？ | 摩擦 |
| 组合构建模型 | 综合三方意见，每只股票持多少？ | 裁判 |
| 执行模型 | 怎么把目标仓位变成真实成交？ | 落地 |
| 数据 | 一切的原材料 | 燃料 |
| 研究 | 打造和改进以上所有模块 | 引擎 |

## 本章的两个关键洞察

### 1. 模块之间是制衡关系

只有 Alpha 模型的系统是危险的：它天然倾向于满仓压在最看好的几只票上（风险模型缺位）、频繁换仓追逐每一个信号（成本模型缺位）。**新手回测只盯收益率，恰恰是把防守两件套拆掉了**——这也是"回测很好、实盘很惨"的常见结构性原因。

### 2. 结构是通用的，实现千差万别

同一张图既能描述大奖章（高频统计套利）也能描述 AQR（低频因子投资）——差别全在每个框的"填法"。这给了你一把解剖刀：**看到任何量化产品，按图索骥问七个问题（每个模块一个），它的轮廓就出来了**。这正是第 12 章评估框架的雏形。

## 一个容易被跳过的细节：模块的时钟

各模块的运行频率不同：Alpha 模型可能每天更新预测，风险模型的因子协方差每周或每月估计，成本模型的参数相对静态，研究则以月和年为单位迭代。**"黑箱"不是一个程序，是一组不同节奏齿轮的咬合**——工程上，模块间的接口和更新时序是无数事故的来源（对照名人堂八卦篇的骑士资本事故）。

## 对应本平台

AlphaForge 是这张图的教学级实现，对照关系值得记住：

- 因子注册中心（`research/factors.py`）= Alpha 模型；
- top-N 等权 + 基准对比 = 组合构建 + 最朴素的风险控制；
- cost 参数 = 简化版成本模型；
- 「因子分析」「因子对比」页 = 研究环节的工具化；
- AKShare → PostgreSQL 管线 = 数据模块。

> **动手**：打开因子对比跑一次海选——你正在操作的就是图左上角"研究 + Alpha"两个框。之后每读一章，回平台找到对应的按钮或参数。

## 本章带走的三句话

1. 七个模块各司其职：进攻、防守、摩擦、裁判、落地、燃料、引擎；
2. 模块互相制衡——只看收益率的回测等于拆掉防守；
3. 结构通用、实现各异：这张图是解剖一切量化产品的手术刀。
